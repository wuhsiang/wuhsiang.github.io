使用GitHub託管學術論文的若干設置

2015-05-12
Elegant Writing
Comments

採用GitHub的private repository來託管正在進行的學術論文是絕佳的選擇，這樣每次的更改都會被記錄，而且還可以通過GitHub的記錄來查看自己有沒有偷懶哦！

很多時候更新文檔時候，有些東西想保留但是卻有影響新文檔的文氣。此時即應該將其刪去，當需要的時候再從之前的commits裡面尋找即可。

Git流程

關於Git使用流程，較好的教程有：

  • GitHub Flow
  • A successful Git branching model及其漢譯版

前者GitHub Flow認為develop分支完全不必要，因此直接在master分支上部署所有的內容即可。

後者A successful Git branching model則採取了一般的觀點，即認為存在三種分支：主分支(master)，開發分支(develop)，和支援分支(feature, release, hotfix)。

大致說來，始終存在的兩個分支是：

  • master: 當develop分支上的代碼達到穩定的狀態時，就可以合併回master分支，從而發佈版本。因此，每次有變化被合併到master分支時，根據定義就是一次新的產品版本發佈。
  • develop: 反應開發的進程。

此外，還有一些有限壽命的分支，這些分支被稱為支援分支，且最終都會被合併入develop或者master分支之後，再被刪除。支援分支主要分為：

  • 特性分支(feature branch)： 也被成為topic分支，源自develop分支，而最終也被合併回develop分支。feature branch為即將發佈的版本開發新功能。
  • 發佈分支(release branch)：源自develop分支，而最終被合併回master和develop分支。release branch為新的產品版本發佈提供準備和支持。一般命名為：release-*。
  • 熱補丁分支(hotfix branch)：源自master分支，而最終被合併回master和develop分支。hotfix branch用於處理未預期的重大bug時，進行快速的產品bug修復。一般命名為：hotfix-*。

學術論文的Git寫作流程

Repository含有單個研究專案

由於涉及正在進行的研究，因此需要付費購買私有倉庫(private repository)。最理想的情形自然是為每個project建立單獨的private repository。此時宜採取GitHub Flow的方案，略去develop分支。直接使用master分支和feature分支來進行部署。

Repository含有多個研究專案

由於已經完成的、正在進行的、打算進行的研究加起來會比較多，因此可能需要採用折中的方案。個人使用的是micro plan，也就是可以最多建立5個私有倉庫，每年支付84美元（教育郵箱可以獲得兩年的免費micro plan）。因此，我並沒有為每個project建立單獨的private repository，而是將類似主題的研究全部放到一個private repository裡面。有時候，一個私有倉庫裡面可能有四五個具體的研究專案。這時候，就需要有寫作流程的計劃了。

參照一般的Git開發流程，我採用了以下方案：

  • master branch: 用於保存穩定的研究專案狀態。換言之，用於保存已經出版了的研究專案。
  • develop branch: 用於保存已經接收的研究專案。一旦研究完成校稿，則將其併入到master branch。
  • feature branch: 用於某個特定的研究專案寫作。例如在private repository裡面同時有五個具體的研究專案，那麼可以分別從develop branch當中分離出來成為單獨的feature branch，並且命名為project-*。
  • release branch: 較少使用。
  • hotfix branch: 用不上。

設置忽略文件

在使用GitHub的時候，也需要進行一些特殊的設置。

原則上講，所有中間過程中生成的臨時文件都最好被忽略掉。例如，在使用LaTeX寫作的時候，就會生成很多臨時文件。

忽略的文件類型可以在.gitignore中進行設置，具體方法可參閱：https://help.github.com/articles/ignoring-files/。由於在windows下無法直接創建沒有文件名的.gitignore文件，所以需要在Git Bash中進入專案目錄，并通過以下命令行創建：

touch .gitignore

然後，可以發現新建了一個.gitignore文件。接下來可以在文件中進行如下設置：

.aux .blg
.dvi .log
.synctex.gz -eps-converted-to.pdf

於是，就可以忽略LaTeX的臨時文件啦。

此外，可能很多臨時文件已經被跟蹤。此時，可以採用命令從repository中刪除臨時文件，如：

git rm -f *.dvi

更多詳細內容可以參見Git基礎。

偶爾會發現一些好玩的網站，因此便在此記錄下來。同時，也會不斷更新啦！

stickk：目標承諾與完成

在網站stickk上，你可以設置瘦身、堅持鍛煉、戒煙或其他目標，然後設定期望的目標達成日期。好玩地是，你可以設置一些懲罰措施，例如沒有完成預定目標，則捐獻10美元給慈善組織甚至非慈善組織。通過目標劃分和懲罰措施，可以激勵自己完成承諾的目標。此外，還可以邀請別人來監督自己，從而使得目標達成更加有動力。以下是stickK FAQ中的說明：

stickK works by providing people with the tools they need to fight back against human nature. By asking users to define their goal (whatever it is!), pick a timeline to accomplish it, and put something at stake (whether it’s money or their reputation), stickK allows users to create a custom plan-of-action for how they’re going to turn their goal into a reality.

So whether you’d like to lose weight, quit smoking or spend more time with your grandkids, stickK is the go-to goal-setting tool for you.

FutureMe：寫信給未來的自己

在網站FutureMe上，你可以給未來的自己寫信，然後信件會在制定的日期裡面寄到你的郵箱裡面。在特別低落的時候，或者對未來有所期待的時候，不妨給未來的自己寫封信哦！

書評：青年王陽明

2015-05-08
Book Review
Comments

書目信息

著述者：杜維明
書名：青年王陽明—行動中的儒家思想
出版社：上海三聯書店
ISBN：978-7-108-04323-8

內容概要

本書探尋王陽明龍場悟道之前的心路歷程，即試圖從王陽明1472-1509人生前三十七年的經歷和心理層面，探討王陽明思想的形成梗概。

全書分為四個部分：

  • 探尋：講述王陽明的家世及早年經歷
  • 抉擇：講述王陽明在陽明洞內的生存選擇
  • 大悟：講述龍場大悟的前後事件
  • 意義：闡述陽明思想及與宋儒思想的傳承發展

陽明先生出生於書香世家，但世代躬耕，以求明哲保身。其父步入仕途，高中狀元，累遷至正三品。陽明先生六歲始能講話，后表現出驚人的語言能力。兩次會試落第，後終於二十七歲進士及第，在京城作為低級官員處理瑣碎事務。

三十歲時，陽明先生以病辭官，此時為第一次精神危機。在陽明洞中，陽明先生試圖割斷對父親與祖母的感情，以求真正意義上的修道生活。然而，年譜報告說：經過長久的思想鬥爭，他終於認識到，對父母的感情深深地扎根於人的本性之中。因此，他通過內心經驗認識到了孝親的不可割捨性，從而接受了儒學的基本信條：自我實現應該接受人類的基本感情，把它當做出發點。如果違背人類的基本情感，自我的基礎就會動搖，自我實現也會變得曲折艱難。基於原初的親情，陽明先生批評了釋家試圖切斷家庭紐帶的做法。相反，他認識到：家庭紐帶並不是嚴格意義上的束縛，而會提供一個導向圖，個人通過它確立自己的恰當位置；它是一個框架，個人在此框架中認清自己的身份。所以，陽明洞的抉擇可以概括為：人生的真正意義應該在人與人的關係中去尋找。在別處尋求個人自由，不僅是逃避，而且違反人性。陽明先生與道家氣質也十分接近，根據其涉入道家的修養，最終發現并高揚曾點精神。重返京城之後，王陽明與湛若水訂交，立志共同傳播新的儒學思想。此時，陽明先生也正式開門收徒。

陽明先生旋即因反對宦官劉瑾，被關押約兩個月。爾後廷杖四十，流放貴州西部的龍場。在異族雜居的龍場，陽明先生經歷了第二次精神危機。在陽明洞抉擇之後，他立志“用世”，即通過授徒來傳播儒學思想。然而在龍場，幾乎與世隔絕，又如何做聖賢呢？在此之前，陽明試圖通過在政治領域或者教育領域有所作為來做聖賢。然而，在龍場則不得不放棄這種路徑。在此背景下，便出現了龍場大悟：聖人之道，吾性自足，向之求理于事物者誤也。對他來說，“如何做聖賢”就是如何通過內心的力量達到自我實現。因此，格物之功，只在身心上做，決然以聖人為人人可到，便自有擔當了。在龍場大悟之後，陽明先生與土著苗人以及地方官府的關係均有了極大地改善。也有諸多學子遠赴龍場，聽陽明先生的講習。

有趣的是，雖然陽明思想與陸象山更為接近，但其受象山影響卻十分微小，反而受朱熹影響極大。陽明與朱熹的重要區分之處在於重新確立了主體性原則。在朱熹思想中，格物是將抽象的道德原則與具體的道德處境聯繫起來的必要環節（唐君毅先生語）。而陽明則創造性地闡述了主體性原則，消除了朱熹的格物所包含的主體與客體的二分，從而將朱熹的格物從一系列零碎的自我控制活動轉變為內部自我更新的連續過程。

評述

作者杜維明先生作為新儒家第三代的傑出代表人物，其作品自然值得一讀。然書中心理層面的分析並不純熟，似可說明杜維明先生在完成博士論文時在心理分析方面的造詣尚淺。

MarkdownPad 2當中數學公式的顯示

2015-04-17
Elegant Writing
Comments

進行寫作時，Markdown和Latex均是較好的方案。前者的優勢在於可以集中精力進行寫作，而後者的優勢在於排版。至於Word嘛… Word沒有任何優勢，直接拋棄好了。

Markdown旨在進行輕量級運作，然而當進行學術論文寫作的時候，經常是需要輸入數學公式的。這時，就需要配合MathJax來使用了。

本地顯示方案

經過嘗試本地以及在線MathJax解決方案之後，發現Markdown Pad 2當中很難很好地集成MathJax。這也意味著，當使用Markdown Pad 2進行寫作時，是無法實時預覽數學公式的效果，只能看到源代碼。

據Markdown Pad上FAQ Forum的資訊來看，開發者似乎有意在下一版本中加入內置MathJax的支持。所以，也只有拭目以待了。

另外，可以在RStudio中採用R Markdown來進行寫作，從而可以一鍵生成預覽html文檔。

在線顯示方案

雖然本地端尚無較好的顯示方案 ，但是如果採用線上寫作方式，那麼問題也就迎刃而解了！

目前最好的online editor應該是StackEdit，可以完美地實時預覽數學公式。

此外，Three ways to use Markdown with Evernote提供了在Evernote中使用Markdown的方法。特別是第一種方法中提到的Markdown Here，可以配合郵件使用。這樣的話，就可以在郵件中進行數學公式寫作啦！

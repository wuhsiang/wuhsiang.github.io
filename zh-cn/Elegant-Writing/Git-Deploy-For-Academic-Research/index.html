使用GitHub托管学术论文的若干设置

2015-05-12
Elegant Writing

採用GitHub的private repository来托管正在进行的学术论文是绝佳的选择，这样每次的更改都会被记录，而且还可以通过GitHub的记录来查看自己有没有偷懒哦！

很多时候更新文档时候，有些东西想保留但是却有影响新文档的文气。此时即应该将其删去，当需要的时候再从之前的commits里面寻找即可。因此，使用GitHub进行学术论文版本控制是十分安全且高效的。

Git流程

关于Git使用流程，较好的教程有：

  • GitHub Flow
  • A successful Git branching model及其汉译版

前者GitHub Flow认为develop分支完全不必要，因此直接在master分支上部署所有的内容即可。

后者A successful Git branching model则採取了一般的观点，即认为存在三种分支：主分支(master)，开发分支(develop)，和支援分支(feature, release, hotfix)。

大致说来，始终存在的两个分支是：

  • master: 当develop分支上的代码达到稳定的状态时，就可以合併回master分支，从而发佈版本。因此，每次有变化被合併到master分支时，根据定义就是一次新的产品版本发佈。
  • develop: 反应开发的进程。

此外，还有一些有限寿命的分支，这些分支被称为支援分支，且最终都会被合併入develop或者master分支之后，再被删除。支援分支主要分为：

  • 特性分支(feature branch)： 也被成为topic分支，源自develop分支，而最终也被合併回develop分支。feature branch为即将发佈的版本开发新功能。
  • 发佈分支(release branch)：源自develop分支，而最终被合併回master和develop分支。release branch为新的产品版本发佈提供准备和支持。一般命名为：release-*。
  • 热补丁分支(hotfix branch)：源自master分支，而最终被合併回master和develop分支。hotfix branch用于处理未预期的重大bug时，进行快速的产品bug修复。一般命名为：hotfix-*。

学术论文的Git写作流程

Repository含有单个研究专案

由于涉及正在进行的研究，因此需要付费购买私有仓库(private repository)。最理想的情形自然是为每个project建立单独的private repository。此时宜採取GitHub Flow的方案，略去develop分支。直接使用master分支和feature分支来进行部署。

Repository含有多个研究专案

由于已经完成的、正在进行的、打算进行的研究加起来会比较多，因此可能需要採用折中的方案。个人使用的是micro plan，也就是可以最多建立5个私有仓库，每年支付84美元（教育邮箱可以获得两年的免费micro plan）。因此，我并没有为每个project建立单独的private repository，而是将类似主题的研究全部放到一个private repository裡面。有时候，一个私有仓库裡面可能有四五个具体的研究专案。这时候，就需要有写作流程的计划了。

参照一般的Git开发流程，我採用了以下方案：

  • master branch: 用于保存稳定的研究专案状态。换言之，用于保存已经出版了的研究专案。
  • develop branch: 用于保存已经接收的研究专案。一旦研究完成校稿，则将其併入到master branch。
  • feature branch: 用于某个特定的研究专案写作。例如在private repository裡面同时有五个具体的研究专案，那麽可以分别从develop branch当中分离出来成为单独的feature branch，并且命名为project-*。
  • release branch: 较少使用。
  • hotfix branch: 用不上。

设置忽略文件

在使用GitHub的时候，也需要进行一些特殊的设置。

原则上讲，所有中间过程中生成的临时文件都最好被忽略掉。例如，在使用LaTeX写作的时候，就会生成很多临时文件。

忽略的文件类型可以在.gitignore中进行设置，具体方法可参阅：https://help.github.com/articles/ignoring-files/。由于在windows下无法直接创建没有文件名的.gitignore文件，所以需要在Git Bash中进入专案目录，并通过以下命令行创建：

touch .gitignore

然后，可以发现新建了一个.gitignore文件。接下来可以在文件中进行如下设置：

.aux .blg
.dvi .log
.synctex.gz -eps-converted-to.pdf

于是，就可以忽略LaTeX的临时文件啦。

此外，可能很多临时文件已经被跟踪。此时，可以採用命令从repository中删除临时文件，如：

git rm -f *.dvi

更多详细内容可以参见Git基础。

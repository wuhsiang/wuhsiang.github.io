Hexo部博客中展示数据分析

2015-09-02
Elegant Writing

使用Hexo可以快速部署博客，并将部落格託管在GitHub上。部落格写作在Markdown编辑器中进行，可以完全专注写作本身。因此，Hexo绝对是目前为止最好的部落格平台。然而除了一般的文本之外，我们有时还需要展示一些数据分析的原理、算法和例子。此时便涉及到了数学公式、代码以及图表。当然，我们不愿意先得到分析结果，再从外部Ctrl + C/V到部落格正文。作为重度强迫症患者，我们希望沿袭文学编程思想，将代码和文字集中在同一个文件裡面。在此情形下，又该如何方便进行写作以及Hexo配置呢？

本文介绍在Hexo部落格中数据分析的高效写作及方便展示的方法：

  • 配置：进行MathJax公式支持的配置
  • 写作：在RStudio中採用R Markdown进行原始文档写作
  • 运行：采用Knitr package中的knit命令生成包含统计结果的markdown文件
  • 微调：进行细微调整

配置：hexo-math

基本的Hexo并不支持Markdown中公式的显示。因此，需要安装hexo-math并进行简单配置来支持公式显示。配置过程分为三步：

首先，安装hexo-math:

$ npm install hexo-math --save

其次，在Blog文件夹中进行初始化：

$ hexo math install

最后，在_config.yml文件中添加plugin:

plugins:
    - hexo-math

如此，便配置好了MathJax环境。虽不能尽如人意，但还是可以大体正常显示数学公式。

写作： R Markdown源文件

进行统计分析的时候，当然首选RStudio编辑器下的R Markdown格式。文字部分遵循通常的Markdown语法，而R代码块内部的内容则遵循通常的R语法。因此，写作十分方便。

运行：Knit

为了得到统计分析结果，只需要採用Knitr package中的knit命令来生成对应的markdown文件即可。当然，也可以直接在RStudio里面点击Knit HTML并且勾选keep markdowm source file以生成Markdown文件。

微调

最后，微调生成的Markdown文件。主要包括两个部分：

  • 修复显示错误的数学公式：由于hexo-math并不完美，所以才有这一步。
  • 更改图片目录：将生成的图片统一複製到Blog文件夹的/source/img/中，并更改Markdown文件中对应的图片路径。

好啦，大功告成！快来试试吧！

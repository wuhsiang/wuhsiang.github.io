Hexo部落格中展示數據分析

2015-09-02
Elegant Writing

使用Hexo可以快速部署部落格，并將部落格託管在GitHub上。部落格寫作在Markdown編輯器中進行，可以完全專注寫作本身。因此，Hexo絕對是目前為止最好的部落格平台。然而除了一般的文本之外，我們有時還需要展示一些數據分析的原理、算法和例子。此時便涉及到了數學公式、代碼以及圖表。當然，我們不願意先得到分析結果，再從外部Ctrl + C/V到部落格正文。作為重度強迫症患者，我們希望沿襲文學編程思想，將代碼和文字集中在同一個文件裡面。在此情形下，又該如何方便進行寫作以及Hexo配置呢？

本文介紹在Hexo部落格中數據分析的高效寫作及方便展示的方法：

  • 配置：進行MathJax公式支持的配置
  • 寫作：在RStudio中採用R Markdown進行原始文檔寫作
  • 運行：採用Knitr package中的knit命令生成包含統計結果的markdown文件
  • 微調：進行細微調整

配置：hexo-math

基本的Hexo並不支持Markdown中公式的顯示。因此，需要安裝hexo-math并進行簡單配置來支持公式顯示。配置過程分為三步：

首先，安裝hexo-math:

$ npm install hexo-math --save

其次，在Blog文件夾中進行初始化：

$ hexo math install

最後，在_config.yml文件中添加plugin:

plugins:
    - hexo-math

如此，便配置好了MathJax環境。雖不能盡如人意，但還是可以大體正常顯示數學公式。

寫作： R Markdown源文件

進行統計分析的時候，當然首選RStudio編輯器下的R Markdown格式。文字部分遵循通常的Markdown語法，而R代碼塊內部的內容則遵循通常的R語法。因此，寫作十分方便。

運行：Knit

為了得到統計分析結果，只需要採用Knitr package中的knit命令來生成對應的markdown文件即可。當然，也可以直接在RStudio裡面點擊Knit HTML並且勾選keep markdowm source file以生成Markdown文件。

微調

最後，微調生成的Markdown文件。主要包括兩個部分：

  • 修復顯示錯誤的數學公式：由於hexo-math並不完美，所以才有這一步。
  • 更改圖片目錄：將生成的圖片統一複製到Blog文件夾的/source/img/中，并更改Markdown文件中對應的圖片路徑。

好啦，大功告成！快來試試吧！
